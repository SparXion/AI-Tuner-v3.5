# AITuner v5.0 — Onboarding Narrative Script
**Date:** March 15, 2026
**Purpose:** Voice and copy for every onboarding screen
**Status:** First draft — John to wordsmith
**Note:** This is the coaching layer. Every screen is a moment
in a conversation, not a UI state.

---

## Design Principles for This Script

- The app speaks in first person plural: "we'll help you"
  not "the app will" or "you should"
- Every screen has one job. One idea. One next step.
- The user is never wrong. They're just earlier in the journey.
- Celebrate every action. Small wins build confidence.
- Never introduce a feature before the user has a reason to want it.
- The coach steps back as the user steps forward.

The founder's name and signature makes it human.
This is not a corporate product. It came from curiosity.

The coach knows what comes next.

## Screen 6: What AI Actually Is

**[Educational. Light. Feels like a smart friend explaining something.]*

---

**Label:**
"Before we go further"

**Headline:**
"Here's the thing nobody
explains about AI."

**Body:**
"Every AI system has a default personality.

Not because of magic. Because someone — a team of engineers
and designers at a company — made decisions about how it
should behave. How confident it should sound. How much it
should explain. Whether it should ask questions or just answer.

Those decisions made sense for most people most of the time.

But you're not most people. You're you.

And the interesting thing — the thing that surprised me
when I first discovered it — is that those defaults
aren't locked. They're starting points.

Every major AI system will adjust its behavior
if you tell it how you want it to behave.

That's what prompts do. That's what you just did.
You told it what you wanted. It listened."

**CTA:**
"Got it — what's next? →"

---

**Coaching note:**
This screen demystifies AI without being condescending.
It validates the user's experience (what they just did worked)
and reframes AI from mysterious black box to
configurable tool that responds to them.
The tone is "smart friend who knows this stuff"
not "tutorial explaining things slowly."

# NARRATIVE PRINCIPLES — SUMMARY

For anyone writing copy for this product going forward:

1. **The app is a coach, not a tool.**
   It guides. It celebrates. It steps back when the user is ready.

2. **The user is never behind.**
   They're exactly where they are. There's no wrong pace.

3. **Features appear when they're needed, not before.**
   The radar shows up after the user has felt what it measures.
   The decoder shows up when the user finds a prompt they want to understand.

4. **Plain English is non-negotiable.**
   If a label requires explanation, the label is wrong.

5. **The origin story is always available.**
   Devashish's prompt. The Instagram scroll. The curiosity.
   This story gives the product humanity and credibility.
   It should be accessible from the about screen at all times.

6. **Transparency is a feature.**
   Self-reported model data is labeled as self-reported.
   Observed behavior is labeled as observed.
   We don't pretend to be objective. We show our sources.

7. **The goal is graduation.**
   Every coaching interaction is working toward
   the moment the user doesn't need the coach anymore.
   That's not failure. That's the whole point.
