Business Rules — New & Updated
R1 — new Clustering threshold: Levers cluster when all members are pointing the same direction (all ≥ 7 or all ≤ 3) AND spread between highest and lowest value ≤ 3 points. Below threshold: cluster splits, each lever generates its own instruction.
R2 — new Dead zone: Levers set 4–6 generate no instruction. Exception: if a lever is part of a cluster and the cluster as a whole is directional, include it in the synthesis even if its individual value is mid-range.
R3 — new Conflict resolution hierarchy: Output (1) → Character (2) → Voice (3) → Thinking (4). When pillars conflict, lower number wins. Hierarchy declaration always appended to generated prompt — never omitted.
R4 — new Source map always generated: Every prompt generation produces a source map simultaneously. The source map is stored alongside the prompt and used to render the "How this was built" UI section. A prompt without a source map is invalid.
R5 — new Persona bypass: Personas provide pre-synthesized instructions and bypass the clustering engine. The persona's lever_profile is recorded in the source map for transparency but does not pass through clustering logic.
R6 — new Knowledge/Behavior separation: KNOWLEDGE_TEMPLATE and INSTRUCTIONS_CONTENT are always separate documents. The system must never combine them into a single output. Mixing them is the primary cause of AI instruction failure.
R7 — new Native behavior labeling: NATIVE_BEHAVIOR always shows two fields separately: self_reported (from elicitation prompts) and observed (from calibration tool). Never combined into a single description. Source always labeled in UI.
R8 — updated Prompt target length: 50–150 words. Above 150 words the model treats content as noise. The clustering and dead zone rules exist specifically to keep prompts within this range. Word count stored in SOURCE_MAP.
R9 — carried forward Tier is one-way: Tier only increases, never decreases. skip_onboarding bypasses tier checks but does not set tier. Config source is immutable after creation.
R10 — carried forward Calibration never auto-applies: CLI calibration tool writes to v5-models.proposed.js only. Human review required before promotion to live. NATIVE_BEHAVIOR updated only after human approval.
